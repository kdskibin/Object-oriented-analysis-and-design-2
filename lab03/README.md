# Парсер PDF на основе паттерна «Цепочка обязанностей»

Веб-приложение на Flask, которое загружает PDF-документ и постранично обрабатывает его содержимое, автоматически выбирая подходящую стратегию (обработчик) для каждой страницы.

---

## Паттерн «Цепочка обязанностей» (Chain of Responsibility)

**Идея:** у нас есть несколько обработчиков, каждый из которых умеет работать с определённым типом страниц.  
Они выстроены в цепочку. Когда приходит запрос на обработку страницы, мы по очереди опрашиваем обработчики на возможность распарсить страницу. Первый обработчик, вернувший True будет использоваться для парсинга.

В проекте это реализовано следующим образом:

- **`BaseHandler`** – абстрактный класс с методами `can_handle(context)` и `handle(context, page_num, output_dir)`.
- Три конкретных обработчика:
  - `SimpleTextHandler` – страницы с чистым текстом.
  - `IntermediateHandler` – страницы со смешанным содержимым (текст, таблицы, изображения).
  - `ImageOnlyHandler` – fallback-обработчик, который рендерит страницу как картинку, если ни один другой не подошёл.
- **`ParsingChain`** получает список обработчиков и для каждой страницы последовательно вызывает `can_handle()`, пока не будет найдена подходящая стратегия.

---

## Преимущества такого подхода

### Гибкость и расширяемость
Добавить новый способ обработки можно без изменения существующего кода.  
Достаточно написать новый класс-обработчик и вставить его в цепочку – например, `FormulaHandler` для страниц с большим количеством математических формул.

### Разделение ответственности
Каждый обработчик знает только о своём типе страниц и не зависит от других.  
Логика не перемешивается в одной гигантской функции с десятками `if-elif-else`.

## Код без паттерна

Так могла бы выглядеть одна "универсальная" функция обработки страницы, принимающая все возможные решения внутри себя:

```python
def process_page(page, page_num, output_dir):
    # Пытаемся понять, что за страница перед нами
    has_images = bool(page.images)
    text = page.extract_text() or ""
    has_broken_text = "(cid:" in text
    tables = page.find_tables()

    if not has_images and not has_broken_text and text.strip():
        # Простой текст
        save_text(text, page_num, output_dir)
    elif len(page.images) <= 10 or len(text) >= 100:
        # Смешанный контент
        # ... много кода для извлечения таблиц, картинок, подписей ...
        images_data = extract_images(page, ...)
        tables_data = extract_tables(page, ...)
        captions = extract_captions(page, ...)
        # ... ещё строки логики ...
        save_mixed_content(text, images_data, tables_data, page_num, output_dir)
    else:
        # Рендерим страницу как картинку
        save_snapshot(page, page_num, output_dir)
```
### Проблемы такого подхода:

- Логика быстро становится громоздкой и трудночитаемой.
- Добавление нового типа страниц требует изменения этого большого метода.
